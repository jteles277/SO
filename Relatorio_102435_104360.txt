Neste Relatório explicaremos o processo de construção do programa netifstat.sh.

Main     
    Como Organizamos os dados recebidos a partir do comando "ifconfig -a":

        criamos 3 arrays um para o tx rx e interfaces
        a partir do command ifconfig -a usando grep, awk, tr -d (deleta o : )
        Organizamos um array (i_data) com 3 colunas (kinda like a matriz) e com linhas iguais ao numero de interfaces)
        exemplo:
            eth0 0 0
            lo 0 0
            wifi0 0 0

        dps usamos o sleep 
        fazemos exatamente a mesma coisa e guardamos tudo noutro array (data)
        exemplo:
            eth0 2 10
            lo 6 5
            wifi0 7 3

    como fazemos a subtração dos dois codigos recebidos

        Fazer a subtração entre os valores dos arrays e salvamos na mesma variavel (data)


    como corremos os argumentos para selecionar as opções do user
        corremos um for loop que percorre todos os argumentos e usamos um switch casa para indentificalos
            -c  muda o valor da variavel boleana de modo que na proxima iteração leio o Regex intruduzido pelo User
            -p  muda o valor da variavel boleana de modo que na proxima iteração leio o numero de interfaces a mostrar
                so efectuado apos o sorting para que as interfaces sejam escolhidas por ordem alfabetica(default) ou pela ordem escolhida pelo User
            -b/k/m
                O b o k e o m são opçoes que mudam o byte_divisor, uma variavel que usamos para para dividir no final para printar a data 
                na forma desejada
            -r  Seleciona o uso da função de sorting SortRX atravez de uma variavel boleana explicada abaixo 
            -t  Seleciona o uso da função de sorting SortTX atravez de uma variavel boleana explicada abaixo 
            -R  Seleciona o uso da função de sorting SortRX atravez de uma variavel boleana explicada abaixo pois caso a interface tenha o Rx maior tbm vai ter o maior R_Rate
            -T  Seleciona o uso da função de sorting SortTX atravez de uma variavel boleana explicada abaixo pois caso a interface tenha o Tx maior tbm vai ter o maior T_Rate            
            -v  Seleciona o uso da função de sorting Reverse atravez de uma variavel boleana explicada abaixo 
            -l  Seta a variavel boleana para ser feito o loop


    Printar a tabela
        Printar o cabeçalho 
            dependendo do valor da variavel loop, o cabeçalho será
                NETIF         TX        RX     TRATE     RRATE
                ou então em caso de loop
                NETIF         TX        RX     TRATE     RRATE     TXTOT     RXTOT
        Printar os valores 
            Iteramos pela data criando a variavel TX_rate e RX_rate dividindo o tx e o rx pelo sleep time;
            Dividimos tudo pelo byte_divisor que será diferente dependendo da opção Selecionada pelo User
            Depois printamos a informação
            exemplo
                eth0           0         0         0         0
                eth1           0         0         0         0


Looping 
        Caso a variavel boleana loop esteja ativa o codigo da main será iterado a parte da selação e opçoes que é so feita uma vez
        Serão usados outros 2 arrays tot_tx e tot_rx cada elemento corresponde a uma interface ordenada de forma alfabetica sendo somados a ele mesmo da iteração anterior,
        dps são inseridos na data e dps são printados numa nova formatação da tabela
            NETIF         TX        RX     TRATE     RRATE     TXTOT     RXTOT
            eth0           0         0         0         0         0         0
            wifi0          0         0         0         0         0         0 


Functions
    SortRX e SortTX
        Usamos um selection sort dependendo do TX ou do RX 
        O selection sort funciona comparando cada elemento i do array com os elementos seguintes posicionando o mais 
            mais alto para cima ate acabar o array
    Reverse
        função que inverte o nosso array trocando os valores dos indexes com a mesma distancia ao centro
    Regex
        Itera por todas as interfaces da data e se a interface corresponder ao Regex desejado vai desta forma
        inseri-la num array intermedio para dps dar de novo set ao array data
